# GIT016: List Format Issues in Commit Body

## Error

List items in the commit body don't follow proper Markdown formatting.

## Why This Matters

- Lists need blank lines before them in Markdown
- Improper formatting breaks rendering in GitHub/GitLab
- Consistent formatting improves readability

## How to Fix

Add empty lines before lists:

```bash
# Wrong:
git commit -sS -m "feat(api): add endpoints
Changes:
- Added user endpoint
- Added auth endpoint"

# Correct:
git commit -sS -m "feat(api): add endpoints

Changes:

- Added user endpoint
- Added auth endpoint"
```

## Markdown Rules

1. Blank line after the title
2. Blank line before any list
3. Consistent list marker (`-`, `*`, or `1.`)
